# 03 - Commercial

**Propósito**: como a Orkestra.AI fecha contrato que se paga. Política comercial, comissão, ICP, autorização de desconto.

## O que mora aqui

- `[[Commission Policy]]` — regras do motor de comissão (base, splits, clawback, carência).
- `[[Discount Authorization Matrix]]` — quem aprova qual nível de desconto.
- `[[ICP - Ideal Customer Profile]]` — quem é cliente bom, quem não é.

## O que **não** mora aqui

- Dashboard comercial ao vivo → ORKESTRA `/commercial/dashboard`.
- Pipeline individual → ORKESTRA CRM + `[[Dashboard - Weekly Review Board]]`.
- Incidentes comerciais (contrato perdido, clawback) → `08 - Learnings`.

## Ritmo de atualização

- `[[Commission Policy]]` só muda com `Decision` em `09 - Decisions`. É contrato interno, não texto vivo.
- `[[Discount Authorization Matrix]]` revisado trimestralmente junto com `[[North Star Metrics]]`.
- `[[ICP - Ideal Customer Profile]]` revisado após cada `Learning` de cliente perdido/quebrado.
