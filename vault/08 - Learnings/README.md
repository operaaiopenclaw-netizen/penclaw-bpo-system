# 08 - Learnings

**Propósito**: lição capturada a partir de um acontecimento real — alerta, incidente, variância de reconciliação, contrato perdido, feedback de cliente.

Aqui vale o princípio #9: **aprendizado tem prazo**. Learning sem próximo passo em ≤ 30 dias vira fofoca.

## O que mora aqui

Uma nota por lição com:

- O que aconteceu (fato, não opinião)
- Por que aconteceu (causa raiz — use os 5 "por quês" se precisar)
- O que fazer a respeito (próximo passo com `[[Decision]]` ou `[[SOP]]` alvo)
- Prazo para fechar o loop (≤ 30 dias)

## O que **não** mora aqui

- Lamentação sem próximo passo ("isso foi ruim" sem ação).
- Dados operacionais brutos — link para ORKESTRA.
- SOPs formais — aqui fica a origem, o SOP mora em `10 - SOPs`.

## Convenção de nome

```
Learning - <descrição curta do aprendizado> YYYY-MM.md
```

Uma Learning nunca é apagada — se virou obsoleta, atualizar status para `closed` com resultado.

## Ritmo

- Abrir em ≤ 72h do acontecimento.
- Revisar semanalmente no Weekly Review — todo Learning `open` ou `in_progress` entra na pauta.
- Fechar com uma de três saídas:
  - **Decision** registrada em `09`
  - **SOP** criado ou atualizado em `10`
  - **Feature** ou parâmetro mudado no sistema (link para commit/PR)
